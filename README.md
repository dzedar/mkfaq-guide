# mkfaq-guide

## 项目简介

`mkfaq-guide` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常以 FAQ、指南或说明文档的形式呈现，内容涵盖不同主题，彼此之间无强关联。仓库本身不针对任何特定域名或网站，仅作为静态 HTML 资源的集合与分发点。

本仓库的主要目的是提供一个结构清晰、便于检索的 HTML 页面存档空间，方便用户直接通过浏览器访问或引用其中的内容。

## 目录说明

```
mkfaq-guide/
├── index.html          # 仓库入口页面，列出所有归档页面索引
├── pages/              # 主要页面存放目录
│   ├── example-1.html  # 示例页面 1
│   ├── example-2.html  # 示例页面 2
│   └── ...
├── assets/             # 资源文件（CSS、图片、字体等）
│   ├── css/
│   ├── images/
│   └── ...
└── README.md           # 本文件
```

- `pages/`：所有独立 HTML 页面均存放于此目录下，文件名应简洁、语义化。
- `assets/`：存放页面所需的静态资源，如样式表、图片等，保持与页面文件的引用一致。
- `index.html`：提供导航功能，便于快速定位到各个页面。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可脱离仓库环境直接运行（需保证资源路径正确）。
- 页面内容不涉及任何具体网站、商业推广或诱导性信息，仅作为知识性或说明性文档。
- 归档页面可能不定期更新、修正或移除，请以仓库当前状态为准。
- 如需引用或分发某个页面，建议直接使用 GitHub Pages 提供的原始链接，或自行克隆仓库后托管到其他静态服务。

## 维护说明

- 本仓库由维护者定期整理和更新，不接受外部直接提交的页面内容（如有建议可提交 Issue 讨论）。
- 所有页面应保持 HTML 语法规范，避免使用过时标签或私有扩展。
- 资源文件应尽量压缩，避免大体积文件影响加载速度。
- 若某个页面不再适用或存在错误，维护者将及时删除或替换，并在 `index.html` 中更新索引。

## 使用方式

1. 直接通过 GitHub Pages 访问本仓库的 `index.html` 即可浏览所有页面列表。
2. 也可克隆仓库到本地，用浏览器打开任意页面文件查看。
3. 若需部署到自己的服务器，请将整个仓库目录上传，并确保静态资源路径正确。

## 许可

本仓库中的页面内容遵循知识共享许可协议（具体版本详见各页面底部声明），资源文件如无特殊说明均采用相同许可。允许自由复制、分发，但需保留原始作者信息及许可声明。
